About This Document
===================

The purpose of this document is to provide information about Loom and it's functionalities for administrators in IT operations and also for users who use Loom for provisioning clusters based on teamplates created by administrators. This document also covers topics of operational management of Loom system. 

Intended audience
=================

This document is intended to be a reference for administrators within IT operations team and also with developers (users). 
The document presumes you are well familiar with Hadoop technologies - specifically Zookeeper. Further it will presume 
knowledge in REST & Ruby.
